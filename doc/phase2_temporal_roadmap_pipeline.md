# Phase 2 Temporal Roadmap Pipeline

This phase implements the first working end-to-end nonholonomic temporal roadmap planner on top of the sampled reusable roadmap from Phase 1.

## Query Pipeline

1. Build or load a static-valid sampled nonholonomic roadmap.
   - Nodes are `(x, y, theta)` poses.
   - Nodes are sampled independently of the query start and goal.
   - Edges are generated by rolling out motion primitives.
   - Edges are kept only if the full sampled trajectory is statically valid.

2. Prepare a query graph.
   - Clone the base roadmap.
   - Add temporary start and goal nodes.
   - Connect start to nearby roadmap/goal nodes with primitive rollouts.
   - Connect roadmap/start nodes to the temporary goal with primitive rollouts.

3. Run temporal A*.
   - Search labels contain `node_id`, `arrival_time`, `cost_to_come`, parent label, and incoming edge.
   - Expansion uses outgoing roadmap edges.
   - The edge departure time is the current label arrival time.
   - The child arrival time is `arrival_time + edge.duration_s`.

4. Validate edges online against dynamic obstacles.
   - Static validity is still checked along the trajectory.
   - Dynamic obstacles are moving circles with constant velocity.
   - Each trajectory sample receives a timestamp.
   - If any sample collides with a moving obstacle at its timestamp, the edge is skipped.

5. Reconstruct the path.
   - The result includes node ids, poses, edge ids, action labels, arrival times, segments, total cost, and traversal time.

The base roadmap is cloned per query before temporary start/goal nodes are added, so the sampled roadmap can be reused across multiple queries without mutation.

## Method Boundary

The main Phase 2 method is:

```text
Sampled Nonholonomic Temporal Roadmap with Online Validation
```

The older primitive-expansion roadmap remains useful as a temporal graph-search baseline, but it is not the PRM-style proposed-method builder.

## Dominance Rule

The first pruning rule is:

```text
best_cost[(node_id, time_bin)]
```

where:

```text
time_bin = round(arrival_time / time_bin_size_s)
```

If a new label reaches the same `(node_id, time_bin)` with cost greater than or equal to the best known cost, it is discarded.

## What Is Intentionally Not Implemented Yet

- wait actions
- offline temporal edge-validity interval precomputation
- uncertainty-aware dynamic inflation
- Gazebo execution
- reactive replanning baseline internals
- large experiment suite

# Implementation Process

These files document how the project was planned and implemented. They are
useful for code understanding, progress tracking, and explaining engineering
choices, but they are not the cleanest final-report text.

## Project-Level Plans

- `nonholonomic_temporal_roadmap_project_plan.md`  
  Original project plan and high-level motivation.

- `nonholonomic_ta_prm_spec.md`  
  Detailed sampled nonholonomic TA-PRM-style planner spec.

- `core_method_completion_and_evaluation_spec.md`  
  ACTEA implementation and evaluation checklist.

## Phase / Architecture Notes

- `refactor_phase1_architecture.md`  
  Phase 1 code architecture and baseline preservation notes.

- `sampled_nonholonomic_phase1_status.md`  
  Sampled nonholonomic roadmap builder status.

- `phase2_temporal_roadmap_pipeline.md`  
  First temporal roadmap query pipeline and online validation flow.

## How to Use These Files

Use this folder when you need to answer:

- what was implemented first
- what each phase added
- why the code is structured this way
- what tasks were originally planned
- what baselines were intended

For final report prose, prefer `doc/paper_content/`.

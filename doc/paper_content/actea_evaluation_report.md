# ACTEA Method and Evaluation Report

## 1. Project Goal

This project studies nonholonomic motion planning in dynamic environments.
The robot is not allowed to move holonomically; it follows motion primitives
generated by a differential-drive model. Therefore, both position and heading
matter. A path that looks short in XY space may still be infeasible if the
robot cannot reach the required heading through available primitives.

The main problem is:

```text
Given a reusable nonholonomic roadmap and known moving circular obstacles,
how can we answer many start-goal temporal planning queries efficiently?
```

The project's proposed method is:

```text
Sampled Nonholonomic Temporal Roadmap with ACTEA
```

where:

```text
ACTEA = Analytic Continuous-Time Edge Annotation
```

The core idea is to move expensive temporal edge-validity reasoning from query
time into reusable edge-level temporal metadata.

## 2. Why This Problem Is Hard

In a static roadmap, each edge only needs to answer one question:

```text
Is this edge geometrically collision-free?
```

In a dynamic world, the same edge has a time-dependent answer:

```text
Is this edge valid if the robot departs at time t0?
```

The same roadmap edge may be:

- valid at `t0 = 2.0s`
- blocked at `t0 = 3.0s`
- valid again at `t0 = 4.5s`

This means edge validity depends on departure time. In hard scenes with many
moving obstacles, online temporal validation may need to repeatedly evaluate
many edge-time pairs during search.

ACTEA addresses exactly this bottleneck.

## 3. Proposed Method

The roadmap is built once using sampled nonholonomic states:

```text
node = (x, y, theta)
edge = statically valid nonholonomic motion primitive rollout
```

During query, the temporal planner searches over labels:

```text
(roadmap_node_id, arrival_time)
```

The key addition is ACTEA. For each roadmap edge and dynamic-obstacle scenario,
ACTEA computes:

- exact blocked departure-time intervals
- exact valid departure-time intervals
- the annotation horizon
- optional per-obstacle blocked intervals

The edge-level metadata is represented by `EdgeTemporalAnnotation` and is
stored in an `EdgeTemporalAnnotationStore`.

The planner exposes formal edge-time APIs:

```python
edge_is_valid_at_time(edge, departure_time_s, ...)
edge_cost_at_time(edge, departure_time_s, ...)
```

The current time-dependent edge cost is:

```text
cost(edge, t0) = edge.geometric_cost    if edge is valid at departure time t0
cost(edge, t0) = infinity              otherwise
```

This makes the method clean to describe as a temporal roadmap method rather
than only a cache optimization.

## 4. What ACTEA Computes

For one piecewise-linear segment of a primitive edge, the robot center is
modeled as:

```text
r(s) = r0 + vr s
```

A moving circular obstacle is:

```text
o(t0 + s) = o0 + vo (t0 + s)
```

Collision occurs when:

```text
||r(s) - o(t0 + s)|| <= R
```

where `R` is the robot radius plus obstacle radius plus clearance.

ACTEA analytically projects this continuous-time collision condition onto the
departure-time axis. The output is a set of blocked intervals and valid
intervals for each edge.

Important scope:

ACTEA is exact for the current piecewise-linear interpolation of sampled
primitive trajectories. It is not claiming to symbolically solve the original
continuous differential-drive dynamics before sampling.

## 5. Compared Methods

The experiments compare these methods.

| Method | Role |
| --- | --- |
| `static_astar` | Negative control / sanity baseline |
| `reactive_replanning` | Stronger reactive baseline using repeated static planning |
| `expansion_temporal` | Weaker temporal graph baseline built by seed expansion |
| `sampled_temporal_online` | Same sampled roadmap, online temporal validation |
| `sampled_temporal_bin_cache` | Same roadmap, sampled time-bin cache |
| `sampled_temporal_actea` | Proposed ACTEA method |

The main comparison focuses on:

- `reactive_replanning`
- `expansion_temporal`
- `sampled_temporal_online`
- `sampled_temporal_actea`

The repeated-query experiment additionally includes:

- `sampled_temporal_bin_cache`

The purpose is to separate three questions:

1. Does sampled temporal roadmap planning work better than weaker baselines?
2. Does ACTEA preserve online validation correctness?
3. Does ACTEA reduce repeated-query temporal validation cost?

## 6. Experiment 1: ACTEA Correctness

### Purpose

This experiment checks whether ACTEA interval lookup agrees with brute-force
online temporal validation.

For many sampled edges and many departure times, the experiment compares:

```text
ACTEA interval membership result
vs.
online exact temporal validation result
```

### Result

| Metric | Value |
| --- | ---: |
| Agreement rate | `1.0` |
| False blocked | `0` |
| False free | `0` |

### Interpretation

This supports the claim that ACTEA is not an approximate shortcut under the
project's trajectory model. It gives the same validity decision as online
validation, but stores the result as reusable edge-time metadata.

The correct conclusion is:

```text
ACTEA matches online temporal validation under the same piecewise-linear
trajectory model.
```

The claim should not be overstated as exactness for the original continuous
robot dynamics before primitive sampling.

## 7. Experiment 2: Repeated-Query Scaling

### Purpose

This is the most important experiment for ACTEA.

The goal is to test a multi-query setting:

- same sampled roadmap
- same dynamic obstacle scenario
- many different start-goal queries
- query counts up to `N = 200`

This is where ACTEA should help. ACTEA has an offline annotation cost, so it
does not need to beat online validation on a single query. It should become
useful when many queries reuse the same annotated roadmap.

### Result

| Method | Build | Annotation | Avg Query | Total over N=200 | Success | Interval Hits | Cache Hits |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| sampled ACTEA | `5.4342` | `3.4182` | `0.2519` | `59.2248` | `1.00` | `741320` | `1765` |
| sampled bin cache | `5.0843` | `0.2521` | `0.2435` | `54.0383` | `1.00` | `0` | `727719` |
| sampled online | `5.1871` | `0.0000` | `0.4132` | `87.8273` | `1.00` | `0` | `0` |

### Interpretation

ACTEA reduces average query time compared with sampled online:

```text
sampled online avg query: 0.4132s
sampled ACTEA avg query: 0.2519s
```

This is about a 39% reduction in average query time.

Total time over 200 queries also improves:

```text
sampled online total: 87.8273s
sampled ACTEA total: 59.2248s
```

ACTEA saves about 28.6 seconds over this repeated-query workload.

The interval hit count is also important:

```text
ACTEA interval hits: 741,320
```

This shows that ACTEA is not only present in the implementation. The temporal
planner is actually using the edge interval metadata heavily during query.

### What This Proves

This experiment supports the main ACTEA story:

```text
Online validation keeps paying temporal collision-checking cost during query.
ACTEA pays an offline annotation cost, then reuses interval lookup across many
queries.
```

### Important Caveat

The bin-cache method is slightly faster in this particular repeated-query run:

```text
sampled bin cache total: 54.0383s
sampled ACTEA total: 59.2248s
```

This does not invalidate ACTEA. It means the current benchmark also benefits
strongly from repeated time-bin hits. The report should frame bin cache as an
engineering ablation:

```text
The bin cache is a fast sampled approximation/reuse mechanism, while ACTEA is
the analytic continuous-time edge annotation contribution.
```

A stronger ACTEA-favorable setting would use more diverse departure times or a
stricter requirement that validity decisions remain continuous-time exact
rather than bin-based.

## 8. Experiment 3: Hard Dynamic Scenes

### Purpose

This experiment checks whether temporal planning matters in difficult dynamic
environments.

The hard scenes include:

- dense cross traffic
- multi-wave moving bottleneck
- dense longitudinal interactions
- tight corridor with heading constraints
- cluttered multi-corridor map
- mixed-flow intersection

Each scene has multiple moving obstacles and multiple start-goal queries.

### Result

| Method | Success | Avg Traversal Time | Avg Replans | Avg Dynamic Rejects |
| --- | ---: | ---: | ---: | ---: |
| expansion temporal | `0.00` | `0.000` | `0.000` | `55.600` |
| reactive replanning | `0.33` | `18.367` | `18.033` | `0.000` |
| sampled ACTEA | `0.93` | `17.600` | `0.000` | `203.500` |
| sampled online | `0.93` | `17.600` | `0.000` | `204.867` |

### Interpretation

The main result is that sampled temporal roadmap methods are much more robust
than the baselines:

```text
sampled temporal success: 0.93
reactive replanning success: 0.33
expansion temporal success: 0.00
```

This shows that:

- reactive replanning is often too local
- seed-expansion temporal graph has weak coverage
- sampled roadmap reuse gives better global path-choice coverage
- temporal reasoning is necessary in these scenes

ACTEA and sampled online have the same success rate because they use the same
roadmap and the same temporal validity semantics. The difference is not path
success; the difference is query-time cost.

In several hard scene families, ACTEA is faster than online:

| Scene | ACTEA Query | Online Query |
| --- | ---: | ---: |
| moving bottleneck multiwave | `0.277s` | `0.695s` |
| tight corridor heading dense | `0.682s` | `2.911s` |
| cluttered multi corridor | `0.444s` | `0.846s` |

These are the strongest examples for the report because they show scenes where
edge validity depends heavily on departure time.

### What This Proves

This experiment mainly supports:

```text
The sampled temporal roadmap is a stronger dynamic-environment planning
framework than reactive replanning or seed-expansion temporal graph baselines.
```

ACTEA improves the efficiency of this framework by replacing repeated online
edge validation with interval lookup.

## 9. Experiment 4: Roadmap-Scale Ablation

### Purpose

This experiment studies how roadmap size affects ACTEA usefulness.

The regimes are:

- sparse
- medium
- dense
- extra dense

The expectation is not that ACTEA always wins. ACTEA has offline annotation
cost, so it should help most when query reuse is high enough to amortize that
cost.

### Result

| Scale | Method | Nodes | Edges | Annotation | Avg Query | Total Runtime | Success |
| --- | --- | ---: | ---: | ---: | ---: | ---: | ---: |
| sparse | online | `432` | `1576` | `0.000` | `0.542` | `14.029` | `0.48` |
| sparse | ACTEA | `432` | `1576` | `0.397` | `0.124` | `3.955` | `0.48` |
| medium | online | `1672` | `11208` | `0.000` | `0.404` | `14.700` | `1.00` |
| medium | ACTEA | `1672` | `11208` | `2.968` | `0.237` | `13.537` | `1.00` |
| dense | online | `2760` | `21154` | `0.000` | `0.474` | `25.150` | `1.00` |
| dense | ACTEA | `2760` | `21154` | `5.652` | `0.376` | `28.612` | `1.00` |
| extra dense | online | `4608` | `26600` | `0.000` | `0.586` | `32.503` | `1.00` |
| extra dense | ACTEA | `4608` | `26600` | `7.132` | `0.565` | `39.505` | `1.00` |

### Interpretation

ACTEA strongly improves query time in sparse and medium roadmaps:

```text
sparse avg query: 0.542s -> 0.124s
medium avg query: 0.404s -> 0.237s
```

In dense and extra-dense roadmaps, ACTEA still lowers average query time, but
the offline annotation cost is larger. Therefore, total runtime may be worse
unless the number of repeated queries is high enough.

This is an important honest finding:

```text
ACTEA is not a free single-query speedup. It is a reusable annotation method
whose benefit depends on the roadmap size and the number of future queries.
```

This actually strengthens the report because it clearly identifies the regime
where the proposed method is useful.

## 10. Experiment 5: Heuristic Ablation

### Purpose

This experiment checks whether the improved heuristic changes the conclusion.

The two heuristic modes are:

- `euclidean`
- `goal_distance_heading_time_lb`

This ablation is intentionally separate from the ACTEA-vs-online comparison.
The main comparison uses the same heuristic for both online and ACTEA.

### Result

| Heuristic | Method | Avg Query | Total Runtime | Success | Expanded Labels |
| --- | --- | ---: | ---: | ---: | ---: |
| euclidean | online | `0.4742` | `28.6426` | `1.00` | `32376` |
| euclidean | ACTEA | `0.2539` | `21.2851` | `1.00` | `32376` |
| heading/time lower bound | online | `0.4663` | `28.5508` | `1.00` | `29785` |
| heading/time lower bound | ACTEA | `0.2560` | `21.3930` | `1.00` | `29785` |

### Interpretation

The improved heuristic reduces expanded labels:

```text
32376 -> 29785
```

This is about an 8% reduction.

However, it does not improve path quality in the current experiment. The saved
raw results show:

```text
euclidean path length: 7.2257
heading/time heuristic path length: 7.5398
```

So the correct conclusion is:

```text
The heuristic modestly improves search effort, but it does not make the route
better in this experiment.
```

The key point is that ACTEA remains faster than online under both heuristics.
Therefore, ACTEA's advantage is not caused by the heuristic.

## 11. Overall Findings

### Finding 1: ACTEA preserves online validation correctness

ACTEA agrees with online validation in the correctness experiment. This
supports using interval lookup as a substitute for repeated online edge
validation under the same trajectory model.

### Finding 2: ACTEA improves repeated-query efficiency

In the repeated-query experiment, ACTEA reduces average query time from
`0.4132s` to `0.2519s` compared with sampled online validation.

The total runtime over 200 queries drops from `87.8273s` to `59.2248s`.

### Finding 3: sampled temporal roadmaps are stronger than reactive and expansion baselines

In hard scenes:

```text
sampled temporal methods: 0.93 success
reactive replanning: 0.33 success
expansion temporal: 0.00 success
```

This shows the value of a reusable sampled roadmap with temporal search.

### Finding 4: ACTEA's benefit depends on amortization

ACTEA has offline annotation cost. It is most useful when:

- there are many queries
- dynamic obstacles cause many edge-time validity checks
- the same roadmap and obstacle scenario are reused
- exact continuous-time validity is preferred over sampled bin reuse

It is less beneficial when:

- there are too few queries
- the roadmap is extremely dense but query count is low
- simple bin-cache reuse already captures most repeated checks

### Finding 5: heuristic helps search effort, not route quality

The improved heuristic reduces label expansions but does not improve path
length in the current result set.

## 12. What Can Be Claimed in the Final Report

Strong claims supported by results:

1. ACTEA matches online temporal validation under the same piecewise-linear
   trajectory model.
2. ACTEA reduces query-time cost in repeated-query settings.
3. Sampled temporal roadmap methods outperform reactive replanning and
   expansion temporal graph baselines in hard dynamic scenes.
4. ACTEA is especially meaningful as a reusable temporal roadmap annotation
   method, not as a one-shot planner trick.

Claims to avoid:

1. Do not claim ACTEA always beats every cache method. The bin cache is faster
   in the current repeated-query table.
2. Do not claim ACTEA always reduces total runtime for every roadmap scale.
   Dense roadmaps need enough repeated queries to amortize annotation cost.
3. Do not claim the heuristic improves route quality. The current path length
   does not support that.
4. Do not claim exactness for unsampled continuous robot dynamics. The exactness
   is for the implemented piecewise-linear trajectory model.

## 13. Recommended Report Narrative

A concise final-report narrative could be:

```text
We propose ACTEA, an analytic continuous-time edge annotation layer for a
sampled nonholonomic temporal roadmap. Instead of validating dynamic obstacle
collisions online for every edge-time pair, ACTEA precomputes blocked and valid
departure-time intervals for roadmap edges. The resulting planner preserves the
same validity decisions as online temporal validation under the implemented
piecewise-linear primitive model, while enabling cheaper repeated queries
through interval membership lookup.

Experiments show that ACTEA matches online validation with 100% agreement in
the tested scenarios. In a repeated-query workload with 200 queries, ACTEA
reduces average query time from 0.4132s to 0.2519s and total runtime from
87.8273s to 59.2248s compared with sampled online validation. In hard dynamic
scenes, sampled temporal roadmap methods achieve 0.93 success rate, compared
with 0.33 for reactive replanning and 0.00 for the expansion temporal baseline.
Roadmap-scale ablations show that ACTEA's benefit depends on amortizing
annotation cost over repeated queries.
```

# ACTEA Method and Evaluation Report

## 1. Project Goal

This project studies nonholonomic motion planning in dynamic environments. The robot is not holonomic; it follows motion primitives generated by a differential-drive model, so both position and heading matter. A path that appears short in XY space may still be infeasible if the robot cannot realize the required heading changes through its available primitives.

The main question of the project is:

```text
Given a reusable nonholonomic roadmap and known moving circular obstacles,
how can we answer many start-goal temporal planning queries efficiently?
```

Our proposed method is:

```text
Sampled Nonholonomic Temporal Roadmap with ACTEA
```

where:

```text
ACTEA = Analytic Continuous-Time Edge Annotation
```

The key idea is to move expensive temporal edge-validity reasoning from query time into reusable edge-level temporal metadata.

---

## 2. Why This Problem Is Hard

In a static roadmap, each edge answers a simple question:

```text
Is this edge geometrically collision-free?
```

In a dynamic world, that is no longer enough. The same roadmap edge can be valid at one departure time, blocked at another, and valid again later. In other words, edge validity becomes a function of departure time rather than a fixed Boolean property.

This creates a computational bottleneck for temporal planning. In hard scenes with many moving obstacles, online planning may repeatedly evaluate many edge-time pairs during search. ACTEA is designed to address exactly this repeated temporal-validation cost.

---

## 3. Proposed Method

The roadmap is built once using sampled nonholonomic states:

```text
node = (x, y, theta)
edge = statically valid nonholonomic motion-primitive rollout
```

At query time, the temporal planner searches over labels of the form:

```text
(roadmap_node_id, arrival_time)
```

This is the core temporal-roadmap pipeline: build a sampled reusable roadmap, clone it per query, connect temporary start/goal nodes, and run temporal A* over roadmap nodes while accounting for time-dependent edge feasibility.

The distinguishing addition is ACTEA. For each roadmap edge and dynamic-obstacle scenario, ACTEA computes reusable temporal metadata:

- blocked departure-time intervals
- valid departure-time intervals
- an annotation horizon
- optional per-obstacle blocked intervals

This metadata is represented as `EdgeTemporalAnnotation` and used through the planner’s edge-time query APIs. The current edge-time cost model is:

```text
cost(edge, t0) = edge.geometric_cost    if edge is valid at departure time t0
cost(edge, t0) = infinity              otherwise
```

This framing is important: ACTEA is not just a cache trick. It is a continuous-time temporal annotation layer on top of a sampled nonholonomic roadmap.

---

## 4. What ACTEA Computes

For one piecewise-linear segment of a primitive edge, the robot center is modeled as linear motion over segment-relative time, and a moving circular obstacle is modeled with constant velocity. Collision is tested by comparing the robot-obstacle distance against the summed collision radius. ACTEA analytically projects this continuous-time collision condition onto the departure-time axis and returns blocked and valid departure-time intervals for the edge.

The scope of the claim must be stated carefully:

> ACTEA is exact for the implemented piecewise-linear interpolation of sampled primitive trajectories.

It is **not** a symbolic exact solution of the original continuous differential-drive dynamics before primitive sampling. This distinction should remain explicit throughout the report.

## 5. Compared Methods

The experiments compare the following methods:

| Method | Role |
| --- | --- |
| `static_astar` | Negative control / sanity baseline |
| `reactive_replanning` | Stronger reactive baseline using repeated static planning |
| `expansion_temporal` | Weaker temporal graph baseline built by seed expansion |
| `sampled_temporal_online` | Same sampled roadmap, online temporal validation |
| `sampled_temporal_bin_cache` | Same roadmap, sampled time-bin cache |
| `sampled_temporal_actea` | Proposed ACTEA method |

The main comparison focuses on:

- `reactive_replanning`
- `expansion_temporal`
- `sampled_temporal_online`
- `sampled_temporal_actea`

The repeated-query experiment additionally includes `sampled_temporal_bin_cache` as an efficiency-oriented ablation. This is important: the bin-cache method is relevant for runtime comparison, but it is **not** the main contribution. The main contribution is ACTEA as a continuous-time annotation method.

## 6. Experiment 1: ACTEA Correctness

### Purpose

This experiment tests whether ACTEA interval lookup agrees with brute-force online temporal validation. For many sampled edges and many departure times, it compares:

```text
ACTEA interval membership
vs.
online exact temporal validation
```

### Result

| Metric | Value |
| --- | ---: |
| Agreement rate | `1.0` |
| False blocked | `0` |
| False free | `0` |

### Interpretation

This is one of the strongest results in the project. Under the same piecewise-linear primitive trajectory model, ACTEA gives exactly the same validity decisions as online temporal validation. In other words, ACTEA does not change the semantics of temporal edge feasibility; it changes how that information is represented and reused.

The correct claim is:

```text
ACTEA matches online temporal validation under the implemented piecewise-linear
trajectory model.
```

The report should avoid overstating this as exactness for unsampled continuous robot dynamics.

## 7. Experiment 2: Repeated-Query Scaling

### Purpose

This is the central experiment for ACTEA. The goal is to evaluate a multi-query regime in which the same sampled roadmap and the same dynamic-obstacle scenario are reused across many different start-goal queries, up to `N = 200`. This is precisely the setting where ACTEA should be valuable, because it pays an offline annotation cost once and then reuses interval lookup across many queries.

### Result

| Method | Build | Annotation | Avg Query | Total over N=200 | Success | Interval Hits | Cache Hits |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| sampled ACTEA | `5.4342` | `3.4182` | `0.2519` | `59.2248` | `1.00` | `741320` | `1765` |
| sampled bin cache | `5.0843` | `0.2521` | `0.2435` | `54.0383` | `1.00` | `0` | `727719` |
| sampled online | `5.1871` | `0.0000` | `0.4132` | `87.8273` | `1.00` | `0` | `0` |

### Interpretation

Compared with sampled online validation, ACTEA substantially reduces repeated-query cost:

- average query time drops from **0.4132s** to **0.2519s**
- total runtime over 200 queries drops from **87.8273s** to **59.2248s**

The interval-hit count is also highly informative:

```text
ACTEA interval hits: 741,320
```

This shows that ACTEA is not merely present in the implementation. It is actively replacing repeated online temporal edge checks with interval lookup at large scale.

### Important Caveat

In this benchmark, the sampled bin-cache method is still slightly faster overall:

```text
sampled bin cache total: 54.0383s
sampled ACTEA total: 59.2248s
```

This should not be hidden. The right interpretation is:

- **bin cache** is a lightweight, highly effective engineering reuse mechanism in this benchmark
- **ACTEA** is the main continuous-time annotation contribution, with stronger correctness and interpretability guarantees

So the strongest supported claim is **not** “ACTEA is always the fastest reuse mechanism.” The stronger claim is:

> ACTEA preserves exact online temporal validity decisions under the implemented model and substantially reduces repeated-query cost relative to online validation.

## 8. Experiment 3: Hard Dynamic Scenes

### Purpose

This experiment tests whether the temporal-roadmap framework remains effective in difficult dynamic environments, including dense cross traffic, moving bottlenecks, longitudinal interactions, and corridor scenes with heading constraints.

### Result

| Method | Success | Avg Traversal Time | Avg Replans | Avg Dynamic Rejects |
| --- | ---: | ---: | ---: | ---: |
| expansion temporal | `0.00` | `0.000` | `0.000` | `55.600` |
| reactive replanning | `0.33` | `18.367` | `18.033` | `0.000` |
| sampled ACTEA | `0.93` | `17.600` | `0.000` | `203.500` |
| sampled online | `0.93` | `17.600` | `0.000` | `204.867` |

### Interpretation

This experiment primarily validates the **sampled temporal roadmap framework**, not ACTEA alone.

The main takeaway is that sampled temporal roadmap methods are far more robust than the weaker baselines:

- `sampled_temporal_actea` and `sampled_temporal_online`: **0.93 success**
- `reactive_replanning`: **0.33 success**
- `expansion_temporal`: **0.00 success**

This supports several conclusions:

- purely reactive replanning is often too local in these scenes
- the seed-expansion temporal graph has weak coverage
- reusable sampled roadmap structure matters
- temporal planning is necessary in these environments

ACTEA and sampled online achieve the same success rate and traversal time because they share the same roadmap and the same temporal feasibility semantics; ACTEA’s role is not to change which path is feasible, but to make temporal edge-use decisions reusable and cheaper to query.

So the hard-scene benchmark should be framed as:

> evidence that the sampled temporal roadmap framework is strong in hard dynamic scenes

not as:

> evidence that ACTEA alone improves path success.

That distinction makes the report much cleaner.

---

## 9. Experiment 4: Roadmap-Scale Ablation

### Purpose

This experiment studies how roadmap size affects ACTEA’s usefulness.

The main question is not whether ACTEA always wins, but **when** its offline annotation cost is amortized effectively. The roadmap-size study therefore identifies the regime in which ACTEA is most useful.

### Interpretation

The roadmap-scale study suggests an important amortization story:

- in smaller or moderately sized roadmaps, ACTEA can significantly reduce query cost
- in very dense roadmaps, annotation cost becomes large, so total runtime may not improve unless the number of future queries is high enough

This is not a weakness in the report. It is actually a stronger and more honest contribution statement:

> ACTEA is a reusable temporal annotation method whose value depends on the interaction between roadmap size, temporal complexity, and repeated-query reuse.

That is a much better conclusion than pretending ACTEA is a free one-shot speedup in every setting.

---

## 10. Secondary Result: Heuristic Ablation

A separate heuristic ablation compares a basic Euclidean heuristic against a stronger heading/time lower-bound heuristic. The result is useful, but it should remain a secondary result rather than a headline contribution.

The main interpretation is:

- the improved heuristic reduces search effort
- it does **not** improve path quality in the current result set
- ACTEA remains faster than online under both heuristics

So this section is best framed as a supporting engineering result, not as a core contribution on par with ACTEA.

## 11. Overall Findings

### Finding 1: ACTEA preserves online validation correctness
ACTEA matches online temporal validation under the same piecewise-linear primitive model. This supports using interval lookup as a substitute for repeated online temporal edge validation.

### Finding 2: ACTEA improves repeated-query efficiency relative to online validation
In the repeated-query experiment, ACTEA reduces average query time from **0.4132s** to **0.2519s** and total runtime from **87.8273s** to **59.2248s** relative to sampled online validation.

### Finding 3: The sampled temporal roadmap framework is stronger than reactive and expansion baselines
In hard dynamic scenes, sampled temporal roadmap methods achieve **0.93 success**, compared with **0.33** for reactive replanning and **0.00** for the expansion-temporal baseline.

### Finding 4: ACTEA’s value depends on amortization
ACTEA is most beneficial when:
- many queries reuse the same roadmap
- dynamic obstacles induce many edge-time validity checks
- exact continuous-time temporal validity is preferable to sampled-bin reuse

It is less beneficial when:
- there are too few queries
- roadmap density is very high but reuse is low
- simple bin-cache reuse already captures most repeated checks efficiently

### Finding 5: Heuristic design helps search effort, not route quality
The improved heuristic reduces label expansions, but the current results do not support claiming a better route. This should stay a secondary observation.

---

## 12. What Can Be Claimed in the Final Report

### Strong claims supported by results
1. ACTEA matches online temporal validation under the same piecewise-linear trajectory model.
2. ACTEA substantially reduces repeated-query cost relative to online validation.
3. Sampled temporal roadmap methods outperform reactive replanning and expansion-temporal baselines in hard dynamic scenes.
4. ACTEA is best understood as a reusable continuous-time temporal annotation method, not as a one-shot planning trick.

### Claims to avoid
1. Do not claim ACTEA always beats every reuse method; bin cache is faster in the current repeated-query benchmark.
2. Do not claim ACTEA always lowers total runtime for every roadmap scale; dense roadmaps need sufficient query reuse to amortize annotation cost.
3. Do not claim the improved heuristic improves route quality; the current results do not support that.
4. Do not claim exactness for unsampled continuous robot dynamics; the exactness is for the implemented piecewise-linear trajectory model.

---

## 13. Recommended Final Narrative

A concise final-report narrative can be:

> We propose ACTEA, an analytic continuous-time edge annotation layer for a sampled nonholonomic temporal roadmap. Instead of validating dynamic-obstacle collisions online for every edge-time pair, ACTEA precomputes blocked and valid departure-time intervals for roadmap edges. Under the implemented piecewise-linear primitive model, ACTEA matches online temporal validation exactly while enabling cheaper repeated queries through interval membership lookup. In a repeated-query workload with 200 queries, ACTEA reduces average query time from 0.4132s to 0.2519s and total runtime from 87.8273s to 59.2248s relative to sampled online validation. In hard dynamic scenes, sampled temporal roadmap methods achieve 0.93 success rate, compared with 0.33 for reactive replanning and 0.00 for the expansion-temporal baseline. These results support ACTEA as a reusable continuous-time temporal annotation method whose benefit emerges most clearly in amortized multi-query settings.
